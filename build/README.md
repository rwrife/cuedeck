# Build resources

Files in this directory are consumed by **electron-builder** as
`buildResources` during packaging (`npm run package`). They are **not** bundled
into the app's `asar`; they are inputs to the installer/icon generation.

## `icon.png` (placeholder)

`icon.png` is a **512×512 placeholder** app icon generated by
[`scripts/gen-placeholder-icon.mjs`](../scripts/gen-placeholder-icon.mjs)
(`npm run icons`). It is deliberately committed as a real PNG so the project can
package on every platform out of the box.

electron-builder derives the platform-specific formats from this single source:

- **Linux** — resized PNGs for the `AppImage` / `.deb` (icon must be ≥ 256×256).
- **Windows** — converted to `.ico` for the NSIS installer.
- **macOS** — converted to `.icns` for the `.dmg` / `.zip`.

### Replacing it with real artwork

Drop a real **1024×1024 (or at least 512×512) square PNG** in as `icon.png`.
electron-builder will regenerate all platform variants automatically. If you want
crisper platform-native assets, you can instead commit `icon.ico` (Windows) and
`icon.icns` (macOS) alongside `icon.png`; electron-builder prefers those when
present.

Do not hand-edit the generated PNG; either replace it wholesale or tweak the
generator script and re-run `npm run icons`.
